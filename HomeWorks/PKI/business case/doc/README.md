# NWTraders – Проектирование системы информационной безопасности

## 1. Исходные данные

Кейс подготовлен на основе бизнес-описания компании NWTraders.

Компания имеет:
- 2 крупных офиса (Москва – 300 сотрудников, Санкт-Петербург – 200 сотрудников)
- 2 магазина (по 25 сотрудников)
- 2 склада (по 5 сотрудников)
- Заготовительные точки в Сибири (от 250 до 1000 сотрудников сезонно)
- Мобильных сотрудников
- Веб-портал для клиентов и партнеров

Инфраструктура:
- Active Directory (Windows Server 2022)
- Один лес, один домен: nwtraders.msft
- Контроллеры домена в обоих офисах
- RODC на крупных удалённых точках
- Microsoft Exchange (Москва и СПб)
- CRM, ERP, бухгалтерские системы
- WEB-портал для клиентов и партнеров
- VPN через Интернет
- Wi-Fi (включая гостевые сети)

---

## 2. Основные риски

1. Утечка конфиденциальных данных (CRM, ERP, финансовая отчетность)
2. Компрометация AD (единый домен)
3. Перехват данных через ненадежные каналы связи
4. Атаки через гостевые Wi-Fi сети
5. Компрометация мобильных устройств
6. Атаки на веб-портал
7. Недостаточная аутентификация пользователей

---

## 3. Архитектурные решения по ИБ

### 3.1 Защита данных при хранении

- BitLocker для рабочих станций и серверов
- Шифрование БД (TDE для SQL Server)
- Разграничение доступа на основе ролей (RBAC)
- Принцип минимальных привилегий
- Разделение административных учетных записей

---

### 3.2 Защита данных при передаче

- IPSec между офисами
- SSL/TLS 1.2+ для веб-портала
- VPN с сертификатной аутентификацией
- Запрет небезопасных протоколов (SMBv1, NTLMv1)

---

### 3.3 Строгая аутентификация

- MFA (2FA) для:
  - VPN
  - OWA
  - Администраторов
  - Веб-портала

- Kerberos внутри домена
- PKI инфраструктура
- Смарт-карты или OTP

---

### 3.4 Сегментация сети

- DMZ (веб-портал)
- Пользовательский сегмент
- Серверный сегмент
- Административный сегмент
- Гостевой Wi-Fi (изоляция)

- NGFW
- IDS/IPS

---

### 3.5 Защита мобильных сотрудников

- MDM (Microsoft Intune)
- Контроль BYOD
- Обязательное шифрование устройств
- Запрет неподписанных приложений

---

### 3.6 Защита Active Directory

- Tier Model (Tier 0 / Tier 1 / Tier 2)
- Мониторинг привилегированных действий
- Регулярный аудит GPO
- Защита контроллеров домена

---

### 3.7 Мониторинг и реагирование

- SIEM (централизованный сбор логов)
- Мониторинг AD, Exchange, VPN, WEB
- SOC-процессы
- Регламент реагирования

---

## 4. Безопасность взаимодействия клиентов

- HTTPS + HSTS
- WAF
- Защита от DDoS
- Сканирование уязвимостей
- Защита от brute-force

---

## 5. Организационные меры

- Единая политика ИБ
- Управление доступом
- Управление уязвимостями
- Резервное копирование
- Обучение сотрудников
- План реагирования на инциденты

---

## 6. Соответствие стандартам

- ISO/IEC 27001
- ISO/IEC 27002
- ГОСТ Р 57580
- PCI DSS (при обработке карт)

## 7. Итог

Архитектура обеспечивает конфиденциальность, целостность, доступность и строгую аутентификацию в распределенной инфраструктуре.
