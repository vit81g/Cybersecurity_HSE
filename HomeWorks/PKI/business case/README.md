# NWTraders – Проектирование системы информационной безопасности (v2)

---

# 1. Архитектура компании

Компания имеет распределенную инфраструктуру:

- Москва (300 сотрудников) – штаб-квартира
- Санкт-Петербург (200 сотрудников)
- Магазины и склады
- Заготовительные точки в Сибири (250–1000 сотрудников)
- Мобильные сотрудники (3G/LTE)
- WEB-портал для клиентов и партнеров

Инфраструктура:
- AD DS (Windows Server 2022), один домен nwtraders.msft
- DC в Москве и СПб
- RODC на удалённых точках
- Microsoft Exchange
- CRM / ERP / Финансовые системы
- VPN через Интернет
- Wi‑Fi (корпоративный + гостевой)

---

# 2. Расширенная модель угроз

## 2.1 Критичные активы

- База данных CRM
- ERP система
- Финансовая отчетность
- Учетные записи AD (особенно Tier 0)
- Почтовая система Exchange
- WEB-портал клиентов
- VPN инфраструктура

---

## 2.2 Основные категории угроз

### Внешние угрозы

1. Фишинг и компрометация учетных данных
2. Атаки на веб-портал (SQLi, XSS, RCE)
3. DDoS
4. Перехват трафика через публичные сети
5. Brute-force на VPN и OWA

---

### Внутренние угрозы

1. Злоупотребление привилегиями
2. Утечка данных сотрудниками
3. Использование несанкционированных устройств
4. Распространение вредоносного ПО внутри сети

---

### Технологические угрозы

1. Компрометация контроллера домена
2. Атаки на RODC в удалённых точках
3. Уязвимости в ERP/CRM
4. Несегментированная сеть
5. Отсутствие контроля Wi‑Fi

---

## 2.3 Возможные сценарии атак

### Сценарий 1 – Фишинг → Компрометация AD

1. Фишинговое письмо
2. Захват учетных данных
3. Доступ через VPN
4. Lateral movement
5. Доступ к файловым серверам

---

### Сценарий 2 – Компрометация WEB-портала

1. Эксплуатация уязвимости
2. Получение shell-доступа
3. Доступ к внутренней БД
4. Утечка данных клиентов

---

### Сценарий 3 – Компрометация мобильного сотрудника

1. Заражение устройства
2. Подключение к VPN
3. Движение внутри сети
4. Распространение малвари

---

# 3. Схема сегментации сети

## 3.1 Логическая сегментация

```
                INTERNET
                    |
                [DDoS Protection]
                    |
                  [WAF]
                    |
                  [NGFW]
                    |
        ------------------------------
        |            |               |
      DMZ        VPN Zone        Public Wi-Fi
        |            |
   WEB Portal    VPN Gateway
        |
   -----------------------------
   |             |             |
User LAN     Server LAN    Admin LAN
   |             |             |
Workstations  AD / ERP     Tier 0 Systems
              Exchange
```

---

## 3.2 Сегменты безопасности

### 1. DMZ
- WEB-портал
- Reverse proxy
- WAF
- Изоляция от внутренней сети

### 2. VPN Zone
- Отдельный сегмент для удаленных подключений
- Обязательная MFA
- Ограничение доступа по ролям

### 3. User LAN
- Рабочие станции
- Ограниченный доступ к серверному сегменту

### 4. Server LAN
- AD
- Exchange
- ERP/CRM
- Базы данных
- Доступ только через межсетевой экран

### 5. Admin LAN (Tier 0)
- Контроллеры домена
- PKI
- Серверы управления
- Jump-серверы
- Полная изоляция

### 6. Guest Wi‑Fi
- Полная изоляция от корпоративной сети
- Только выход в Интернет

# 4. Дополнительные меры защиты

## 4.1 Zero Trust элементы

- Проверка устройства перед доступом
- Conditional Access
- MFA для всех критичных систем
- Проверка состояния устройства (Compliance)

---

## 4.2 Защита Active Directory

- Tier Model (0/1/2)
- Отдельные админ-аккаунты
- PAW (Privileged Access Workstations)
- Мониторинг изменений в AD
- Защита от DCSync

---

## 4.3 Мониторинг

- SIEM
- Сбор логов с:
  - DC
  - Exchange
  - VPN
  - WEB
  - NGFW
- Use Case:
  - Lateral movement detection
  - Privilege escalation
  - Mass data exfiltration
