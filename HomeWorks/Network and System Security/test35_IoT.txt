Тема 35. Безопасность IoT устройств и каналов передачи данных в IoT

1. Какой тип уязвимости присутствует, когда программист не учитывает размер ввода, который может ввести пользователь?

Переполнение буфера

2. Какой тип носителя памяти предоставит место для хранения собранных данных на устройстве IoT?

SD-карта

3. Какой тип атаки использует уязвимости серверов для предоставления неавторизованным пользователям более высоких уровней доступа, чем разрешено?

Эскалация привилегий

4.Какие стандарты беспроводной связи IoT могут использовать производители IoT на больших расстояниях, сохраняя при этом определённый уровень безопасности?
(Возможно несколько вариантов ответа)

LoRa
LTE-M


5. Что подразумевается под термином big.LITTLE?

Технология ЦП, которая использует различные ядра ЦП для обработки задач в зависимости от требований к обработке

6. Какую уязвимость уровня коммуникаций OWASP следует исследовать при защите поверхности атаки сетевого трафика IoT?

Фаззинг протокола

7. Какой тип беспроводной сети IoT будет соединять аудиоустройства и смарт-часы с мобильным телефоном, который служит шлюзом IoT?

Wireless personal-area network

8. В какой атаке субъекты угрозы располагаются между источником и получателем с целью прозрачного мониторинга, перехвата и управления коммуникацией?

Человек посередине

9. Какая настраиваемая беспроводная топология IEEE 802.15.4 может содержать большое количество полнофункциональных устройств и небольшое количество устройств с ограниченными функциями?

cluster-tree

10. Какой вариант беспроводной связи IoT обычно используется устройствами, которые требуют подключения к глобальной сети с низким энергопотреблением и не используют фиксированный источник питания?

LoRaWAN
