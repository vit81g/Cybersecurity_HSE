Тест по Теме 11.  IPv6

1. Основные причины перехода на протокол IPv6 (возможно несколько вариантов):

При использовании IPv6, нет необходимости использования NAT, несмотря на то, что данная возможность поддерживается

При использовании IPv6 нет необходимости использования VLSM, несмотря на то что данная возможность поддерживается

Использование IPv6 решает проблему дефицита IP адресов

2. Какие типы рассылок не поддерживаются в IPv6:

Широковещательные (Broadcast)

3. Выберите верные IPv6 адреса (возможно несколько вариантов):

e80::650a:5f4e:a9fb:f2dc

fe80:0000:0000:0000::2022

2022:64FD:A:1123:FED:ABC:DEF:101

4. Какие утверждения об адресах IPv6 Link-local верны (возможно несколько вариантов):

Адреса Link-local не используются для маршрутизации

Адреса Link-local обычно используют EUI-64 для генерации Interface-ID

5. Выберите наиболее короткий вариант сокращения IPv6 адреса 2021:0BCD:2A42:0000:0000:CAB0:0000:0007:

2021:BCD:2A42::CAB0:0:7

6. Какие утверждения о технологиях сосуществования IPv6 и IPv4 верны  (возможно несколько вариантов):

Туннелирование позволяет обеспечить транзит трафика IPv6 через сети, поддерживающие только IPv4

Большинство современных операционных систем сетевых устройств и серверов/клиентов поддерживают концепцию Dual Stack

7. Системный инженер выполняет проектирование распределенной сети с протоколом IPv6. Требования к протоколу:

1). Масштабируемость. Планируется подключить около 200 филиалов;

2). Широкая поддержка оборудования разных производителей;

3). Учет пропускной способности каналов связи при выборе наилучшего маршрута;

4). Протокол должен принадлежать к классу IGP. 

Выберите протокол маршрутизации соответствующий указанным требованиям:

OSPFv3


8. Какие утверждения, о назначении IPv6 адресов на сетевых устройствах, верны  (возможно несколько вариантов):

При использовании SLAAC, клиент получает только IPv6 адрес, но не маску подсети и основной шлюз

Адреса Link-Local назначаются клиенту как с помощью SLAAC, так и DHCPv6

Статический адрес IPv6 может быть задан полностью администратором, включая префикс и идентификатор интерфейса

При использовании SLAAC, клиент может получить как IPv6 адрес, так и адрес основного шлюза от ближайшего маршрутизатора

При использовании DHCPv6, клиент может получать IPv6 адрес из пула, определенного администратором

Статический адрес IPv6 может быть задан частично администратором. Идентификатор интерфейса назначается администратором, а префикс получен автоматически, используя EUI-64
