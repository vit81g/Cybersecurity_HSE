1. Какие утверждения верны об обработке трафика IP-телефонии SIP для межсетевого экрана? (возможно несколько вариантов)
в случае использования пакетной фильтрации протокола SIP, медиатрафик будет заблокирован

межсетевой экран должен обеспечить передачу трафика протоколов TCP/UDP порт 5060
протокол SIP требует обязательной инспекции SSL/TLS
протокол SIP передаёт отдельные потоки трафика сигнализации и медиатрафика
для передачи трафика IP-телефонии требуется поддержка анализа сообщений SIP на уровне приложений
протокол SIP не поддерживает передачу через межсетевой экран

4. Какие из перечисленных мер позволяют повысить характеристики надёжности сети? (возможно несколько вариантов)

рекомендуется использовать межсетевые экраны в высоко доступной паре (HA pair) или кластере
рекомендованный режим использования межсетевого экрана — установка в режиме «out-of-path», не предусматривающий прохождение трафика через межсетевой экран

межсетевой экран рекомендуется к установке в единственном экземпляре

использование виртуальных межсетевых экранов позволяет сократить время восстановления после отказов


5. Выберите корректные описания инспекции SSL, организованной на межсетевом экране? (возможно несколько вариантов)

пользователь может обнаружить выполнение инспекции SSL с помощью анализа сертификатов, предоставляемых посещаемыми веб-ресурсами

использование SSL-инспекции не оказывает существенного влияния на производительность межсетевого экрана

для использования SSL необходимо использовать доверие к корпоративному PKI или сертификату межсетевого экрана

инспекция SSL обеспечивает динамический подбор ключей симметричного шифрования

инспекция SSL резко поднимает эффективность защиты

инспекция SSL не может быть реализована для некоторых сайтов


6. Какие принципы являются верными в отношении проектирования политики межсетевого экрана? (возможно несколько вариантов)

общие правила необходимо располагать выше частных

наименее популярные правила по количеству срабатываний необходимо помещать в конец политики межсетевого экрана
производительность межсетевого экрана не зависит от количества правил

использовать как можно меньшее количество правил для оптимизации производительности устройства

частные правила необходимо располагать выше общих

8. Какие утверждения о URL-фильтрации верны? (возможно несколько вариантов)

эффективность URL-фильтрации повышается при использовании инспекции HTTPS

URL-фильтрация является автономным модулем защиты и не требует обновления

модуль URL-фильтрации защищает от атак, использующих уязвимости информационных систем

URL-фильтрация может быть выполнена на прокси-сервере или межсетевом экране

URL-фильтрация предусматривает построение политики по категориям веб-ресурсов

10. Какие утверждения о листах управления доступом (ACL) на маршрутизаторах Cisco верны? (возможно несколько вариантов)

ACL может быть использован для фильтрации трафика, трансляции NAT, направления трафика в VPN

расширенные ACL подходят для описания большинства L3 и L4 параметров трафика

стандартные ACL подходят для разграничения доступа к интерфейсам управления маршрутизаторов

инверсная маска 0.0.0.3 соответствует одному IP-адресу

стандартные ACL могут описывать трафик только с использованием адресов назначения

в случае использования ACL на интерфейсе, ACL одновременно контролирует весь трафик, проходящий через маршрутизатор

все ACL, описывающие трафик IPv4, имеют последнюю неявную запись, запрещающую любой трафик
